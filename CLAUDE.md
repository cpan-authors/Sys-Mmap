# CLAUDE.md — Sys::Mmap

## What this is

Sys::Mmap is a Perl XS module wrapping the POSIX `mmap()` system call. It maps files (or anonymous memory) into Perl scalars. POSIX-only — no Windows support.

- **Mmap.xs** — All C/XS code: `mmap()`, `munmap()`, `msync()`, `hardwire()`, `DESTROY()`, `constant()`
- **Mmap.pm** — Perl glue: exports, AUTOLOAD for constants, tied interface (`new`, `TIESCALAR`, `STORE`, `FETCH`)

## Build & test

## XS architecture

### Magic system

Mmap'd SVs carry `PERL_MAGIC_ext` magic with a custom vtable (`mmap_magic_vtbl`). The magic stores a `mmap_info_t` struct with the original `base_addr` and `total_len` (including page-alignment slop). This enables:
- Correct `munmap()` even when the SV pointer is offset from the page boundary
- Automatic cleanup via `mmap_magic_free()` when the SV is destroyed
- Detection of mmap'd SVs via `find_mmap_magic()`

### Key helpers

- `mmap_reset_sv()` — Clears SvPVX/SvCUR/SvLEN/SvOK/SvREADONLY after unmapping
- `find_mmap_magic()` — Walks the magic chain looking for our vtable
- `constant()` — Returns IV values for MAP_*, PROT_*, MS_* constants

### Page alignment

`mmap()` handles non-page-aligned offsets by computing `slop = off % pagesize`, mapping from `off - slop`, and adjusting `SvPVX` by `+slop`. The magic stores the true base address for correct unmapping.

## XS gotchas

- **xsubpp blank line sensitivity**: A blank line followed by column-1 `#if` inside a CODE block is parsed as a function boundary by xsubpp. Never put blank lines before `#if` in CODE blocks.
- **Optional XS parameters**: Parameters after `;` in PROTOTYPE must have default values in the parameter list (e.g., `= 0` for SV*), otherwise xsubpp generates unconditional `ST(N)` reads past the actual argument count.
- **`return` after `croak()`**: Dead code — `croak()` does a `longjmp`. Safe to omit.
- **SvTYPE range checks**: Don't use `SvTYPE(sv) > SVt_PVMG` as an upper bound. In Perl 5.12+, `SVt_PVLV > SVt_PVMG`, so legitimate mmap'd SVs can be rejected. Check for magic first, fall back to type heuristics only for legacy SVs.
- **SvREADONLY persistence**: Sticky across operations. Must explicitly clear with `SvREADONLY_off()` before reuse.
- **Pointer precision**: Never use `NV` (double) for addresses or constants — precision loss above 2^53. Use `UV`/`IV` (`newSVuv(PTR2UV(addr))`).
